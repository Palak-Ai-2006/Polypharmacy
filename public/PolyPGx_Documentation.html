01 / Motivation

The Polypharmacy Crisis

Over 40% of adults aged 65 and older take five or more medications simultaneously. Each additional drug exponentially increases the probability of a dangerous interaction, particularly when genetic variation alters how the body metabolizes those drugs. Current EHR alert systems suffer from alert fatigue, with clinicians overriding more than 95% of all warnings because they lack pharmacogenomic context.

125,000+ Deaths per Year

Adverse drug reactions are the fourth leading cause of death in the United States. The majority stem from preventable drug-drug interactions.

Genetic Blind Spots

Approximately 98% of patients carry at least one actionable pharmacogenomic variant, yet fewer than 1% have ever been tested.

Alert Fatigue

Clinicians override 95% of drug interaction alerts because existing systems generate noise without clinical context.

02 / Scope

What PolyPGx Does

A five-layer clinical reasoning engine combining deterministic pharmacology logic with LLM-powered clinical reasoning to produce actionable, EHR-ready assessments.

CYP Collision Detection

Deterministic engine mapping four major CYP enzymes (3A4, 2D6, 2C19, 2C9) across substrates, inhibitors, and inducers. Zero AI hallucination risk.

Phenoconversion Modeling

Detects when drug inhibitors shift a patient's metabolizer phenotype (e.g., normal to poor), revealing hidden toxicity risks.

RAG-Enriched Reasoning

Retrieves CPIC guidelines and PharmGKB annotations via ChromaDB vector search, feeding evidence into Gemini for grounded reasoning.

EHR-Ready Clinical Notes

Auto-generates Assessment and Plan notes that a hospitalist can paste directly into the electronic health record.

04 / Installation

Getting Started

PolyPGx runs locally with minimal setup. The only hard requirement is a free Gemini API key from Google AI Studio.

Prerequisites

ToolVersion
Node.js18.x or newer
npm9.x or newer
Gemini API KeyFree at AI Studio
ChromaDB (optional)0.4 or newer, for RAG layer

Setup Steps

  1. Clone the repository
    git clone https://github.com/your-org/polypgx.git
  2. Install dependencies
    npm install --legacy-peer-deps
  3. Configure environment
    Copy .env.example to .env.local and add your GEMINI_API_KEY
  4. Start the development server
    npm run dev
  5. Optional: Start ChromaDB for RAG
    pip install chromadb then chroma run --port 8000 --path ./chroma_db
    Ingest: npx ts-node scripts/ingest.ts
Build Note

npm run build may fail under Turbopack due to an upstream ESM/CJS conflict with @chroma-core/default-embed. This does not affect the development server. For production deployment, use Webpack mode.

05 / Usage

Clinical Workflow

Enter a patient's medications and genetic profile. PolyPGx returns a full pharmacogenomic risk assessment in seconds.

  1. Enter Patient Information
    Name, age, and genetic metabolizer phenotypes for CYP3A4, CYP2D6, CYP2C19, and CYP2C9.
  2. Add Medications
    Type drug names with real-time RxNorm autocomplete. Use Ctrl+K to focus the search bar.
  3. Run Analysis
    Click "Analyze Interactions" or load a pre-built demo case. Analysis auto-triggers for demo profiles.
  4. Review Results
    Inspect CYP enzyme collisions, phenoconversion events, per-drug risk cards, and structured LLM reasoning.
  5. Export
    Copy the clinical note to clipboard, or click "Print Report" for a clean PDF.
